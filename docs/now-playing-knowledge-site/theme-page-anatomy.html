theme page anatomy

Purpose

This page documents the internal anatomy of now-playing/theme.html.

It exists because theme.html is not just a simple color picker. It is a client-side shell-theme editor with a token grid, preset controls, import/export tools, save/reset state controls, localStorage-backed persistence, postMessage-based shell synchronization, and editor self-theming behavior.

Main anatomical regions

  1. shell editor header and container
  2. token grid / token card region
  3. preset workflow region
  4. import/export region
  5. save/reset/status region
  6. export modal region
  7. localStorage persistence layer
  8. live shell synchronization layer
  9. editor self-theming layer

Token grid / token card region

Key anchors include #tokenGrid and token card elements created by render(). This is the main live editing surface and the first stop for token-card layout or control changes.

Preset workflow region

Key anchors include #presetSelect, #savePresetBtn, and #deletePresetBtn. Start with loadPresets(), savePresets(), and active-preset logic.

Import/export region

Key anchors include #importBtn and #exportBtn. Start with normalizeThemePayload(...) and buildThemePayload(...).

Save/reset/status region

Key anchors include #saveBtn, #resetBtn, and #statusMsg. This is the explicit persistence/control strip.

Export modal region

Key anchors include #exportModal, #exportJsonText, #exportCopyBtn, and #exportCloseBtn. If the task is about modal UX, textarea formatting, or copy behavior, start here.

LocalStorage persistence layer

Observed keys include nowplaying.themeTokens.v1, nowplaying.themePresets.v1, and nowplaying.themeActivePreset.v1. Start with load(), loadPresets(), and savePresets().

Live shell synchronization layer

Observed message types include np-theme-updated, np-theme-sync, and np-theme-load-preset. Start with push() and window.addEventListener('message', ...).

Editor self-theming layer

Key functions include applyThemeEditorZones(), applyPickerTileBg(), and updateEditorChrome(). If the editor looks wrong while exported tokens are correct, the bug may live here rather than in shell-consumed theme payload logic.

Practical “where do I start?” map

  • Token card layout/controls: start with #tokenGrid and render().
  • Preset behavior: start with preset controls and preset storage logic.
  • Import/export behavior: start with I/O buttons, export modal, and payload normalization/build logic.
  • Save/reset behavior: start with #saveBtn, #resetBtn, #statusMsg, and localStorage write logic.
  • Shell sync: start with push(), message handlers, and the three theme message types.
  • Editor appearance: start with updateEditorChrome(), applyThemeEditorZones(), and applyPickerTileBg().

Current status

This page makes explicit that theme.html is not just a picker. It is a multi-region live design tool with token editing, preset management, import/export, local persistence, shell synchronization, and editor self-theming behavior.
