diagnostics page anatomy

Purpose

This page documents the internal anatomy of now-playing/diagnostics.html.

It is a live diagnostics console with endpoint selection/execution, path/method/body editing, optional track-key injection, curl and response copy helpers, favorites, and embedded live UI surface cards.

Main anatomical regions

  1. shell redirect / page header layer
  2. request runner card
  3. request body / method / path controls
  4. favorites and endpoint selection layer
  5. track-key-aware execution controls
  6. response / copy / curl helper layer
  7. live embedded surface card grid
  8. diagnostics script orchestration layer

Request runner card

The main request-console region is anchored by #endpointFilter, #endpoint, #method, #path, #body, #runBtn, and #status.

Request body / method / path controls

This subregion shapes the outbound request via #method, #path, and #body.

Favorites and endpoint selection layer

Anchored by #endpoint and #favBtn. This region allows filtering endpoints, selecting them, and marking favorites.

Track-key-aware execution controls

Anchored by #useTrackKey. This lets Diagnostics test both open and protected behavior.

Response / copy / curl helper layer

Anchored by #copyBtn, #copyCurlBtn, and the response output/status area.

Live embedded surface card grid

This region contains embedded live cards for surfaces such as now-playing, player, peppy, controller, and kiosk, allowing side-by-side comparison of API truth and visible UI truth.

Diagnostics script orchestration layer

Most behavior appears to be driven by scripts/diagnostics.js, which likely orchestrates endpoint catalog/filtering, favorites, request execution, track-key attachment, curl generation, response copy logic, and embedded-card refresh behavior.

Practical “where do I start?” map

  • Endpoint execution flow: start with the request runner anchors and scripts/diagnostics.js.
  • Favorites: start with #endpoint, #favBtn, and favorite logic in scripts/diagnostics.js.
  • Track-key testing: start with #useTrackKey and request execution logic.
  • Curl/response copy: start with #copyCurlBtn, #copyBtn, and response/curl helper logic.
  • Embedded live cards: start with the card markup in diagnostics.html and refresh logic in scripts/diagnostics.js.

Current status

This is the seventh real page-anatomy reference in the wiki. Its main job is making explicit that diagnostics.html is not just a diagnostics readout, but a multi-region debugging console.
