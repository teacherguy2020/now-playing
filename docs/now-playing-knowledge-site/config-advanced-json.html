config advanced json

Purpose

This page documents the Advanced JSON section of now-playing/config.html.

This is the escape hatch and power-user layer of the Config page: the place where an operator can bypass the structured cards and edit the full config document directly.

Main visible controls

  • advancedJsonDetails
  • fullJson
  • formatJsonBtn
  • saveFullBtn

Full-config textarea

fullJson is populated from the full loaded config as pretty JSON, giving direct access to the complete config object rather than just the parts exposed by structured cards.

Formatting helper

formatFullJson() parses the textarea content as JSON and rewrites it prettified. This is a convenience feature, but it also nudges operators toward valid structured editing.

Full save path

saveFullCfg():

  • requires track key
  • parses the full JSON textarea
  • POSTs to /config/runtime
  • sends { fullConfig: full }
  • marks config saved on success
  • shows the restart modal on success

This is the rawest config persistence path visible in the page.

Relationship to the normal save path

The normal saveCfg() path assembles a config payload from structured controls. The Advanced JSON path instead trusts the operator to edit the whole config object directly.

So the right mental model is:

  • structured cards = safer/default path
  • Advanced JSON = escape hatch / power-user path

Current status

This block is important because it changes the operational power level of Config: full raw config editing, formatting, and full-save behavior all live here.
