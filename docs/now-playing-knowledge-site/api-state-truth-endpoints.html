api state truth endpoints

Purpose

This page documents the most important state/truth endpoints in the now-playing app-host API, especially /now-playing, /next-up, and Alexa-adjacent state helpers.

Why this page matters

These are not just random routes. They are some of the system’s central visible-truth surfaces.

Endpoint: GET /now-playing

/now-playing is one of the most important visible-state endpoints in the system.

Direct source evidence shows:

  • debug=1 mode
  • last-good/stale-response behavior via serveCached(...)
  • fallback to cached payload with _stale and _staleAgeMs
  • 503 behavior if no last-good payload exists
  • heavy mode-aware and radio-aware interpretation logic inside the route

Do not assume it is just raw MPD state or that stale output automatically means a bug.

Endpoint: GET /next-up

/next-up is the closest companion to /now-playing for queue-head truth.

Direct source evidence shows:

  • debug=1 mode
  • special handling for AirPlay
  • moOde status inspection via nextsong / nextsongid
  • MPD playlist lookup by position or id
  • stream/YouTube-aware interpretation for the next item
  • different art handling for file vs stream targets

Do not assume it always has a next item or that AirPlay should behave like local-file queue playback.

Endpoint: GET /alexa/now-playing

This is an Alexa-specific visible-state helper route. Direct source evidence shows freshness/age handling, a derived nowPlaying-like payload, and alexaMode: true based on the remembered alexaWasPlaying object.

Endpoint: GET /alexa/next-up

Direct source evidence shows this route returns a 307 redirect to /now-playing, making the Alexa “next up” semantics intentionally alias the broader visible now-playing truth.

Endpoint: GET /track

/track matters, but it is not the same thing as /now-playing. A good current interpretation is that /track is closer to track-level data lookup, while /now-playing is closer to mode-aware presentation truth.

Core distinctions to preserve

  • /now-playing vs /track: richer presentation truth vs track-level data
  • /now-playing vs /next-up: current visible truth vs upcoming-item truth
  • Alexa helper routes vs normal playback routes: voice-mode continuity/state memory vs general playback state

Suggested starting paths by question

  • Visible current song/art/status wrong: start here, then playback-authority-by-mode, then fragile-behavior-ownership.
  • Next-up behavior wrong: start here, then queue-and-playback-model, then controller-queue-interface.
  • Alexa-visible state wrong: start here, then alexa-interface, then playback-authority-by-mode.

Current status

This page makes the most important state/truth endpoints more explicit, so future agents do not treat /now-playing as just another route in the list.
