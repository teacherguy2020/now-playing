controller kiosk scaffold

Purpose

This page documents now-playing/controller-kiosk.html.

controller-kiosk.html is a direct standalone kiosk/controller scaffold page. It is not the main kiosk launch path.

Important file

  • Primary file: now-playing/controller-kiosk.html
  • Direct endpoints: GET /config/runtime, GET /now-playing, GET /config/diagnostics/queue, GET /config/queue-wizard/playlists

What the page actually contains

Direct file inspection shows a fixed three-pane layout under #app with grid columns 320px 560px 400px: Sources, List, and Now Playing.

Sources pane

#sources contains buttons for library, playlists, radio, podcasts, youtube, and queue. The page also explicitly labels itself 1280x400 kiosk scaffold (phase 1).

List pane

loadList(src) currently has concrete behavior for only two sources: queue loads GET /config/diagnostics/queue, and playlists loads GET /config/queue-wizard/playlists. Other sources render placeholder text saying browsing wiring is next.

Now Playing pane

The pane uses #art, #title, #artist, and #album. refreshNowPlaying() ensures the runtime track key, fetches GET /now-playing, maps title/artist/album fields, resolves artwork from albumArtUrl or displayArtUrl, and falls back to /art/current.jpg. It refreshes every 5 seconds.

Action buttons

The page includes #btnAppend, #btnCrop, and #btnReplace, but they are still placeholder handlers that show alert text like Append action wiring in phase 2.

Runtime key behavior

ensureKey() calls GET /config/runtime and extracts config.trackKey for follow-up requests.

What this page is

  • a direct standalone kiosk/controller scaffold page
  • a self-contained 1280×400 three-pane prototype layout
  • a partial browser-side implementation with real now-playing and list-loading behavior
  • a useful evidence page for earlier kiosk-controller design intent

What this page is not

  • the main kiosk launch bridge (kiosk.html)
  • the main live kiosk shell owner (controller.html in kiosk mode)
  • a thin redirect alias page
  • a fully completed kiosk controller in its current file state

Current status

This page now has a firmer repo-backed definition: controller-kiosk.html is a self-contained kiosk/controller scaffold with real now-playing polling and partial list loading, direct app-host API calls, explicit phase-1 scaffold labeling, and placeholder action wiring. It should be treated as a distinct prototype path, not as the main kiosk runtime.
