integrations

Purpose

This page documents the external-system and cross-boundary integration layer in the now-playing project.

Integrations are not a side topic. They are one of the main reasons the project is operationally complex.

Strong current integration map

1. moOde

moOde is the anchor runtime, not just one integration among many. It is central to browser-target control, display/runtime actions, and visible display effects.

2. MPD

MPD remains foundational to playback and queue behavior even when app-host routes mediate visible behavior.

3. YouTube

YouTube is a first-class integration family with direct route support like /youtube/resolve, /youtube/search, /youtube/playlist, /youtube/proxy/:id, and /youtube/queue.

4. Alexa

Alexa includes both setup/provisioning paths and visible-state helper routes such as /alexa/now-playing, /alexa/next-up, and /alexa/was-playing.

5. Radio metadata and enrichment boundaries

Radio matters because metadata-truth mistakes are easy here. The branch includes explicit eval/debug support and conservative enrichment guardrails.

6. Last.fm / scrobbling / vibe-adjacent integrations

These affect both external activity recording and queue-generation logic.

7. Podcasts

Podcast support is a real integration/runtime family with concrete subscription, refresh, nightly-run, episode, and playlist-building routes.

What this branch is now confident about

  • moOde is the central runtime anchor, not just one integration among peers
  • MPD remains foundational, but app-host logic frequently mediates visible behavior
  • YouTube, Alexa, radio, podcasts, and Last.fm each have real route families and surface-level branch pages
  • many “display” or “playback” problems are actually integration-boundary problems
  • the important operational question is often “which boundary is responsible?” not just “which page looks wrong?”

Current status

This page should now be read as the parent hub for integration-boundary reasoning. It is no longer just a list of outside systems; the current wiki supports the stronger truth that integration boundaries are one of the main structural and operational realities of the project.
