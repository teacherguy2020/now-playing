tablet interface

Purpose

This page describes the tablet-oriented interface family in the now-playing ecosystem.

It exists because tablet usage is not just a resized desktop or mobile experience. The tablet branch appears to be one of the richest controller-facing surfaces in the system, combining browse and discovery flows, queue and playback control, pane-embedded subviews, kiosk-style display behavior, and settings/configuration affordances.

Why the tablet interface matters

Tablet-related surfaces appear to be a major convergence point for controller-family interaction, media-library browsing, kiosk pane routing, visualizer integration, and settings/presentation customization.

The tablet shell also appears to clarify some architectural behaviors more explicitly than the main controller shell, especially around pane messaging and pane state.

Important file

Primary file:

  • now-playing/controller-tablet.html

Related pages already documented include tablet-kiosk-shell-differences.md, kiosk-right-pane-routing.md, embedded-pane-contracts.md, genre-pane-messaging.md, and visualizer-in-embedded-mode.md.

What the tablet interface appears to include

The tablet interface appears to include browse/category surfaces such as playlists, artists, albums, genres, podcasts, and radio; queue access and queue-wizard-related flows; recent-content rails and pane-open interactions; right-pane embedded subviews in landscape or kiosk-like contexts; visualizer access and visualizer-related state handoff; tablet-specific settings surfaces; and kiosk-like display behavior when the tablet shell is used in kiosk-oriented contexts.

Current architectural role

A useful current interpretation is that controller-tablet.html is one of the central orchestration shells of the UI, combining navigation, recent-content browsing, pane hosting, and profile-aware presentation, and supporting both ordinary tablet control usage and kiosk/presentation-adjacent usage.

Likely implementation dimensions

The tablet interface likely spans several implementation dimensions: main shell HTML in controller-tablet.html, embedded child controller pages, pane-routing logic, profile/theme/color handling, recent-content rails and content panes, queue and browse data-loading calls, tablet settings subviews, and visualizer/display-mode handoffs.
