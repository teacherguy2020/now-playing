kiosk designer

Purpose

This page documents now-playing/kiosk-designer.html.

kiosk-designer.html is the kiosk authoring, preview, and push console. It is not the live kiosk shell itself.

Important files

  • Primary file: now-playing/kiosk-designer.html
  • Primary preview/push target: now-playing/kiosk.html
  • Key endpoints: GET /config/runtime, GET /config/moode/browser-url/status, POST /config/moode/browser-url

What the page actually contains

Direct file inspection shows UI elements including #sizeSel, #presetSel, #exportPresetBtn, #importPresetBtn, #pushBtn, #targetHint, #stat, color inputs, and iframe #preview.

Important current facts:

  • #sizeSel currently offers only kiosk.html at 1280×400
  • #preview embeds kiosk.html
  • Save/Delete preset buttons exist in markup but are hidden in script

Key state and storage behavior

persistProfile() writes both nowplaying.kiosk.profile.v1 and nowplaying.mobile.profile.v2, including theme: 'auto', colorPreset, recentSource: 'albums', custom colors, and controller-facing profile fields.

Preview URL construction

buildRel() builds the relative kiosk target, forcing pushRev, theme=auto, recentSource=albums, colorPreset, and custom color query params.

Preview update contract

refreshPreview() reloads the iframe with the rebuilt URL. postLiveTheme() posts type: 'np-live-theme-update' with custom colors into the preview iframe.

Built-in preset behavior

The page defines an in-file THEME_PRESETS table and populates #presetSel from it. Current behavior is built-in preset selection plus import/export JSON, not a general saved-preset subsystem.

Export/import behavior

Export serializes the active preset/color values into JSON and opens the export modal. Import prompts for pasted preset JSON, applies color values, persists profile state, and posts live theme updates into the preview iframe.

moOde target check behavior

ensureKey() fetches /config/runtime and extracts config.trackKey. refreshTargetHint() checks /config/moode/browser-url/status, compares the current moOde target to http(s)://<host>:8101/kiosk.html, and updates #targetHint.

Push-to-moOde action flow

When #pushBtn is clicked, the page ensures the runtime track key, builds the relative kiosk target, turns it into http://<host>:8101/<target>, POSTs it to /config/moode/browser-url with reason kiosk-designer-push, persists the profile locally, and updates #stat.

What this page is

  • the kiosk preview console
  • the kiosk theme/color authoring surface
  • the local profile sync point for kiosk-related state
  • the moOde target verification surface
  • the push-to-moOde control page for kiosk browser-target updates

What this page is not

  • the live kiosk shell
  • the controller-backed kiosk runtime page
  • a generic display-render mode
  • a full saved-preset management system in its current live behavior

Current status

This page now has a firm repo-backed definition: kiosk-designer.html is the authoring, preview, verification, and push surface for kiosk presentation. It previews kiosk.html, persists kiosk/controller-facing profile state, checks moOde browser-target status, and pushes live kiosk URLs into moOde through the app-host API.
