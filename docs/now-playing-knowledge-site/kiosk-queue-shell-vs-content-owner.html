kiosk queue: shell owner vs content owner

Purpose

This page exists to answer a specific recurring question quickly:

If I need to change the queue UI used inside kiosk/right-pane flows, where should I start?

The important answer is that kiosk queue behavior splits into shell ownership and content ownership.

The short answer

  • Queue content UI owner: controller-queue.html / controller-queue-interface
  • Kiosk pane shell owner: controller.html kiosk-mode logic / kiosk-right-pane-routing and controller-kiosk-mode
  • Embedded-mode contract: embedded-pane-contracts

So if the queue looks wrong inside kiosk right pane, you first need to decide whether the bug belongs to the queue UI itself or to the kiosk pane shell around it.

Key proof points

1. Kiosk right-pane routing maps queue into controller queue

kiosk-right-pane-routing shows that when right-pane mode is active:

  • kiosk-queue.html → controller-queue.html
  • the mapped page is loaded into #kioskPaneFrame
  • the parent adds embedded=1 and theme/layout parameters

That is the clearest evidence that the kiosk queue path is not a separate kiosk-only queue implementation.

2. Controller queue is the real queue UI surface

controller-queue-interface states that:

  • controller-queue.html is the main queue inspection/control surface
  • it has both standalone and embedded roles
  • kiosk queue access is controller-backed rather than a kiosk-only duplicate

3. Embedded pane behavior is a real contract

embedded-pane-contracts shows that child pages loaded in the kiosk pane respond to:

  • embedded=1
  • theme and color parameters
  • compact embedded layout rules
  • postMessage-based parent/child behavior such as np-kiosk-hide-pane

So some kiosk queue issues are not really queue-data issues. They are embedded-pane contract issues.

Practical starting paths

If the queue controls or queue list itself are wrong

  1. controller-queue-interface
  2. queue-and-playback-model
  3. route-ownership-map

If the kiosk pane opens the wrong page or behaves wrong as a pane

  1. kiosk-right-pane-routing
  2. controller-kiosk-mode
  3. embedded-pane-contracts

If the embedded queue looks different from standalone queue

  1. embedded-pane-contracts
  2. controller-queue-interface

Do not confuse

  • Kiosk queue shell is not the same thing as queue content UI.
  • Queue interface is not the same thing as Queue Wizard.
  • Kiosk-named route does not automatically mean kiosk-only implementation.

Current status

This page exists mainly to remove synthesis burden. The answer is now explicit:

  • Shell owner: kiosk/controller shell routing and pane lifecycle
  • Content owner: controller-queue.html as the real queue UI surface
  • Embedded behavior owner: the embedded pane contract between parent shell and child page
