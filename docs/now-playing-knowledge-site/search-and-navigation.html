search and navigation

Purpose

This page is the rendered wiki’s dedicated search/start surface.

It exists so an agent or human can begin from one obvious place, run a quick search, and jump into the right page or section without first guessing which hub page to open.

If the task is question-first rather than page-first, start with agent-start.

If the task is about maintaining the wiki itself — ingesting new truth, querying effectively, or linting for health — use wiki-operations.

How to use this page

  • Use the search box to find a page by topic, heading, or summary text.
  • Use the grouped lists when you know the branch but not the exact page.
  • Use the “Best matches” panel to jump directly to a matched section when available.

Best pages by task

  • Need overall orientation? Start with system-overview and architecture.
  • Need to find code ownership? Start with source-map and route-ownership-map.
  • Need operator/runtime truth? Start with config-interface and diagnostics-interface.
  • Need playback/queue truth? Start with queue-and-playback-model and controller-queue-interface.
  • Need display or kiosk behavior? Start with display-interface and kiosk-interface.
  • Need metadata/library cleanup? Start with library-health-interface.
  • Need theme/token behavior? Start with theme-interface.

What the search matches

The rendered quick search currently matches:

  • page labels and titles
  • generated topic keywords
  • short page summaries
  • major section headings

When a heading match is available, result links can jump to the matching section anchor.
