config podcasts and library paths

Purpose

This page documents the podcast- and path-adjacent portions of now-playing/config.html that extend beyond the core Network & Runtime bootstrap card.

The Podcasts block sits at the intersection of feature enablement, API-host filesystem paths, nightly automation, and runtime verification/repair.

Main visible controls

  • featurePodcasts
  • pathPodcastRoot
  • podcastsCronHint
  • podcastsCronInstallHelpBtn

Podcasts feature enablement

The page loads and saves features.podcasts and uses syncPodcastsUi() to keep the section state aligned with whether Podcasts is enabled.

Podcast root path

pathPodcastRoot loads/saves through paths.podcastRoot, defaults to /var/lib/mpd/music/Podcasts, and participates in runtime path verification when Podcasts is enabled.

Nightly automation / cron expectations

The page uses:

  • GET /podcasts/nightly-status

to report whether nightly automation is active, and generates a cron command that calls:

  • POST /podcasts/nightly-run

with x-track-key.

This shows that Podcasts config is tied to a background automation expectation on the API host.

Runtime verification overlap

checkRuntimeEnv() includes the podcast root in its path checks when Podcasts is enabled. If the folder is missing, the UI can offer a repair action.

Podcast-root creation

ensurePodcastRoot() can:

  • prompt for confirmation
  • call POST /config/runtime/ensure-podcast-root
  • rerun runtime verification afterward

That makes the Podcasts block part feature toggle, part environment-repair surface.

Current status

This branch is best understood as:

  • podcast feature enablement
  • podcast-root path ownership
  • nightly automation guidance
  • runtime/path repair hooks
