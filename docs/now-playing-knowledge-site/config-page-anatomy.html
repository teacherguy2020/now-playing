config page anatomy

Purpose

This page documents the internal anatomy of now-playing/config.html.

It is a multi-card operator console with runtime/bootstrap setup, environment verification, feature toggles, integration setup, service controls, database maintenance actions, notification settings, raw JSON editing, and save/reload/restart workflows.

Main anatomical regions

  1. shell redirect / top action strip
  2. runtime/network bootstrap card
  3. podcasts and library-path card
  4. radio artwork and display/render feature card(s)
  5. mpdscribble service-control card
  6. Last.fm / Vibe card
  7. Alexa setup card
  8. ratings maintenance card
  9. track notifications / Pushover card
  10. animated art and album personnel card(s)
  11. advanced JSON editor card
  12. save / reload / restart workflow layer

Shell redirect / top action strip

This layer determines whether the page runs standalone or inside the app shell, and provides page-level save/reload/restart entrypoints such as #saveBtnTop, #reloadBtn, #saveBtn, and #modalRestartBtn.

Runtime/network bootstrap card

This is the foundational Step 1 card for runtime/bootstrap setup. Key anchors include #apiNodeIp, #mpdHost, #moodeSshHost, #configTrackKey, #checkRuntimeBtn, #runtimeCheckStatus, #runtimeCheckDetails, and #sshSetupCmds. Important functions include applyInitialNetworkPrefills(), checkRuntimeEnv(), ensurePodcastRoot(), and refreshSshSetupHelp().

Podcasts and library-path card

Key anchors include #podcastsEnabled, #podcastsRoot, and #podcastsCronEnabled.

Radio artwork and display/render feature card(s)

These cover visual presentation-related feature toggles. Key anchors include #radioArtworkEnabled, #moodeDisplayEnhanced, and display-target check helpers such as checkDisplayTakeoverTargetNow().

mpdscribble service-control card

Key anchors include #mpdscribbleEnabled, #mpdscribbleRefreshBtn, #mpdscribbleStartBtn, and #mpdscribbleStopBtn.

Last.fm / Vibe card

Key anchors include #lastfmEnabled, #lastfmApiKey, #lastfmUser, and #lastfmRowsMode.

Alexa setup card

Key anchors include #alexaEnabled, #alexaPublicDomain, #checkAlexaDomainBtn, and #alexaRouteWebhookUrl.

Ratings maintenance card

Key anchors include #ratingsEnabled, #ratingsDbStatus, #ratingsBackupBtn, and #ratingsRestoreBtn.

Track notifications / Pushover card

Key anchors include #notifyEnabled, #notifyPushoverToken, #notifyPushoverUser, and #notifyAlexaMaxAgeMs.

Animated art and album personnel card(s)

Key anchors include #animatedArtEnabled and #albumPersonnelEnabled.

Advanced JSON editor card

The raw config editor region is anchored by #fullJson, #saveFullBtn, and #formatJsonBtn. The main logic center here is saveFullCfg().

Save / reload / restart workflow layer

This cross-card workflow layer ties the page together. Important functions include loadCfg(), saveCfg(), saveFullCfg(), and restartServicesNow().

Practical “where do I start?” map

  • Runtime/bootstrap behavior: start with the runtime/network anchors and checkRuntimeEnv().
  • Podcast setup behavior: start with the podcast anchors.
  • Display enhancement settings: start with #moodeDisplayEnhanced and display-target check helpers.
  • Last.fm / Vibe: start with the Last.fm anchors.
  • Alexa setup: start with the Alexa anchors and check button.
  • Ratings backup/restore: start with the ratings status/backup/restore anchors.
  • Raw JSON editor: start with #fullJson, #saveFullBtn, #formatJsonBtn, and saveFullCfg().
  • Save/restart flow: start with the save/reload/restart anchors plus saveCfg() and restartServicesNow().

Current status
