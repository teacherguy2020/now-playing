wiki lint and health

Purpose

This page defines what a useful wiki lint/health pass should check in the now-playing knowledge base.

If wiki-operations defines what lint is, this page defines what a good lint pass should actually look for.

Why this page matters

A knowledge base does not stay useful only by growing. It stays useful by remaining reachable, trustworthy, non-duplicative, retrieval-friendly, and explicit about uncertainty.

Core lint question

Does this wiki still help a future human or agent get to the right answer faster and with fewer wrong turns?

What to check during a lint pass

1. Reachability and navigation

  • substantive pages with no meaningful inbound link
  • rendered pages that exist but are not browse-reachable
  • missing hub/branch links
  • missing start-surface links

2. Source/render consistency

  • source updated but rendered lagging
  • rendered updated but source missing the same conceptual link
  • navigation implied but not linked

3. Scope clarity and duplication

  • nearby pages with overlapping scope
  • companion pages not acknowledging each other
  • see-also links hiding stronger structural relationships

4. Retrieval quality

  • recurring questions that still require multi-page synthesis every time
  • places where a smaller model will likely jump wrong
  • missing question-first routing

Recent good retrieval fixes include kiosk-queue-shell-vs-content-owner and queue-interface-vs-queue-wizard.

5. Confidence and proof hygiene

  • prose that sounds too certain for the evidence available
  • inferred structure not marked as still to verify
  • pages that should separate proven facts from interpretation more clearly

6. Search usefulness

  • important retrieval pages missing from start surfaces
  • unclear page titles
  • missing heading anchors where section-jump would help

7. Coverage health

  • branches growing without clear structure
  • repeated references but no dedicated page
  • stale coverage assumptions compared to repo-coverage-notes

Suggested lint cadence

  • Lightweight lint: after a small cluster of additions
  • Branch lint: after a substantial branch expansion
  • Periodic wiki lint: occasionally across the whole wiki

Good outcomes from a lint pass

A good lint pass often produces one missing link fix, one scope clarification, one start-path improvement, one synthesis page, or one uncertainty/proof correction — not necessarily a huge rewrite.
