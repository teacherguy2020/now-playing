config lastfm and scrobbling

Purpose

This page documents the Last.fm and scrobbling-related portions of now-playing/config.html.

It exists because the Config page contains two tightly related but distinct clusters:

  • mpdscribble service control
  • Last.fm / Vibe configuration

Why this page matters

This cluster connects runtime scrobble generation, Last.fm identity/configuration, Vibe/discovery behavior, row presentation choices, and service-control actions.

1. mpdscribble controls

Observed UI elements include:

  • featureMpdscribbleControl
  • mpdscribbleStatus
  • mpdscribbleRefreshBtn
  • mpdscribbleStartBtn
  • mpdscribbleStopBtn

This block governs whether mpdscribble controls are exposed and gives operators direct runtime control over the service.

Important action surfaces

  • GET /config/services/mpdscribble/status
  • POST /config/services/mpdscribble/action

The action endpoint is used for operations like start and stop, and requires the track key.

2. Last.fm / Vibe configuration

Observed UI elements include:

  • featureLastfm
  • lastfmApiKey
  • lastfmUsername
  • lastfmRecentsReplaceRadio
  • lastfmRecentsMode

This block affects whether Last.fm-backed features are enabled, what account/API identity is used, whether Last.fm replaces the Favorite Radio row, what row mode is shown, and whether Vibe tools are meaningfully available.

Important modes

  • toptracks
  • recenttracks
  • topartists
  • topalbums

Important UI hints

  • When enabled/configured, Vibe tools are available in Queue Wizard.
  • The Last.fm row relies on scrobble history.
  • mpdscribble is typical, but not the only possible scrobble source.
  • Required API-host dependencies for Vibe include python3-mpd, python3-requests, and python3-mutagen.

Save/load behavior

Observed config-save/load behavior includes:

  • features.mpdscribbleControl
  • features.lastfm
  • lastfm.apiKey
  • lastfm.username
  • lastfm.recentsReplaceRadio
  • lastfm.recentsMode

The Last.fm API key is also cached locally under nowplaying.secret.lastfmApiKey.

Important distinction

A good current distinction is:

  • mpdscribble = service that may generate scrobble history
  • Last.fm = external service/account + Vibe/discovery feature consumer

So when something breaks, different questions apply:

  • Is the scrobble service running?
  • Are Last.fm credentials configured?
  • Is there enough scrobble history?
  • Are Vibe prerequisites installed?

Current status

This config cluster is not just “put in an API key.” It is:

  • service control
  • feature enablement
  • discovery/Vibe setup
  • row-behavior shaping
  • prerequisite-aware integration config
