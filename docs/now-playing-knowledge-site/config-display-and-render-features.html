config display and render features

Purpose

This page documents the display- and presentation-oriented feature blocks inside now-playing/config.html.

It exists because several Config modules directly affect visible rendering and presentation behavior even though they are smaller than the runtime or ratings blocks.

Current feature clusters in this page include:

  • Radio Artwork
  • moOde Display Enhancement
  • Animated Art
  • Album Personnel

1. Radio Artwork

This is a presentation-facing feature flag, not just backend radio support. It affects how radio playback is visually enriched in now-playing surfaces.

2. moOde Display Enhancement

The UI describes this as enabling Now Playing display enhancement on moOde, making Peppy/Player display tools available, and allowing the moOde target URL to be set to this server.

In the wiki’s terminology, this feature block helps expose:

  • Player display-mode tooling
  • Peppy display-mode tooling, including Peppy skin-oriented tooling where relevant

Observed helper includes checkDisplayTakeoverTargetNow(), which inspects GET /config/moode/browser-url/status and reports whether the moOde target URL is properly set.

3. Animated Art

This is a presentation/rendering feature with obvious user-visible effect. It changes how art appears in live hero/index experiences.

4. Album Personnel

This is another presentation feature with a real environment dependency. It affects visible metadata richness on now-playing surfaces.

Architectural interpretation

A good current interpretation is that these features are the rendering/presentation toggles of Config; they directly shape the visible playback and display experience, and some also include live target verification or host dependency checks.
