config display and render features

Purpose

This page documents the display- and presentation-oriented feature blocks inside now-playing/config.html.

Current feature clusters in this page include:

  • Radio Artwork
  • moOde Display Enhancement
  • Animated Art
  • Album Personnel

Why this page matters

These features are easy to underestimate because their UI is compact, but they influence what users actually see in live playback and display surfaces.

1. Radio Artwork

featureRadio controls radio per-track artwork behavior in now-playing flows.

2. moOde Display Enhancement

featureMoodeDisplayTakeover and displayTakeoverTargetHint govern whether Now Playing display enhancement is enabled on moOde and whether the moOde browser target appears to be pointed at the expected display URL.

Observed active verification includes:

  • GET /config/moode/browser-url/status

This is one of the strongest links between Config and live display-routing behavior.

3. Animated Art

motionArtToggle controls whether animated Apple Music artwork is used when available, affecting hero/index presentation behavior.

4. Album Personnel

featureAlbumPersonnel controls whether the personnel block appears on the Now Playing page, with a host dependency on metaflac.

Current status

These cards are best understood as the rendering/presentation feature toggles of Config: compact in UI, but important in visible effect.
