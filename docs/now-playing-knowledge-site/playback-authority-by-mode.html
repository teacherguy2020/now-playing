playback authority by mode

Purpose

This page documents how playback truth changes depending on the active playback mode.

This is one of the most important operational truths in the project: there is not one universal source of truth for current song, current art, current status, and what should be shown to the user. The authority path depends on the active playback mode.

Core model

Playback authority is mode-dependent. MPD remains important, but it is not the only truth source that matters for visible behavior. App-host routes often translate multiple underlying truths into one visible result.

Mode 1: local file playback

Local file playback is the clearest case where MPD truth and local library/file truth align most directly. Art authority often resolves through moOde/local-file cover-art paths.

Mode 2: AirPlay

AirPlay metadata and art are strongly influenced by the AirPlay metadata path, including aplmeta.txt, AirPlay-specific readers, and moOde current-song/status interpretation for AirPlay.

Mode 3: UPnP

UPnP is its own branch, not a generic stream case. Visible truth can combine moOde status, UPnP-specific resolution logic, and attempts to resolve items back toward local-file truth.

Mode 4: radio / stream

Radio/stream is one of the most nuanced authority paths in the project. Raw stream metadata is often not trusted directly as final truth; visible output can be shaped by holdback, cleanup, enrichment, suppression, station-logo fallback, and cache-backed current-art paths.

Mode 5: unresolved / fallback-driven case

When the system cannot confidently resolve a better authority path, fallback behavior becomes authoritative enough for presentation. Cache-backed current-art paths and last-known-good preservation can become the practical visible truth.

Cross-mode principles

  • Do not assume one universal authority.
  • Visible truth is often app-mediated.
  • Art authority is especially mode-sensitive.
  • Troubleshooting should start with mode classification.

Suggested troubleshooting order by mode

  • Local file: check MPD song/queue, local file identity, and moOde/local-file art path.
  • AirPlay: check AirPlay metadata path, aplmeta.txt, reader health, and local overrides/watchdog behavior.
  • UPnP: check moOde status, UPnP-specific resolution logic, and whether fallback to local-file truth is expected.
  • Radio/stream: check raw metadata quality, holdback logic, cleanup/normalization, enrichment guardrails, and station-logo/fallback art path.
  • Unresolved/fallback: check last-good-render preservation and cache-backed art behavior.
