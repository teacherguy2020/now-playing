queue-wizard internals

Purpose

This page documents the current internal model of Queue Wizard in the now-playing system.

Current evidence already supports a stronger statement: Queue Wizard is a real queue-shaping layer, it sits above raw queue mutation, it uses preview/apply semantics, it is called by substantive controller/now-playing surfaces, and it intersects with Last.fm / Vibe-driven queue creation.

Scope

What this page now treats as established:

  • Queue Wizard is the queue-shaping and queue-curation layer of the system
  • it is not the same thing as the queue page
  • preview/apply is one of its defining patterns
  • album-level and feature-driven queue generation belong here

The remaining gap is the exact route/file/helper contract behind every Queue Wizard action.

Why Queue Wizard matters

Queue Wizard is one of the strongest signs that now-playing is not merely a thin MPD remote. It computes candidate queue changes, previews them, and applies richer queue logic than simple append/play/replace primitives.

High-level role

Queue Wizard is the queue-shaping and queue-curation layer of the system. It sits above raw transport and raw queue mutation and turns feature-side intent into a deliberate queue plan.

Queue Wizard versus raw queue mutation

Raw queue mutation means direct append/replace/crop operations. Queue Wizard adds a higher-order layer on top: previewing queue changes before apply, album-level planning, structured queue shaping, and feature-driven queue generation.

Current known semantics

  • Preview / apply pattern: Queue Wizard computes a candidate queue change, supports preview, then apply commits it.
  • Album-level queue actions: media/library structures become queue plans here.
  • Feature-driven queue shaping: Last.fm/Vibe tools feed into Queue Wizard behavior.

Current known caller surfaces

  • substantive controller-now-playing* pages
  • controller-side queue and now-playing interactions more broadly
  • Last.fm / Vibe discovery tooling

Queue Wizard in the overall playback model

Using the layered model, Queue Wizard is a Layer 4 concept: queue shaping / curation.

Current route knowledge

The wiki already supports the existence of a Queue Wizard route family with preview/apply semantics and album-level actions. The remaining work is exact route names, request/response shapes, and owning file/module chain.

Relationship to YouTube queueing

POST /youtube/queue is direct feature-specific queue mutation. Queue Wizard is the broader queue-shaping and curation layer.

Relationship to embedded queue surfaces

controller-queue.html is queue inspection/control UI. Queue Wizard is the higher-order queue-shaping logic that determines what the queue should become.

Current status

This page now gives the wiki a sharper internal model of Queue Wizard: it is the queue-shaping layer, it uses preview/apply semantics, it is called by substantive controller/now-playing surfaces, it intersects with discovery/curation features like Last.fm/Vibe, and its exact route/file contract is the next deeper proofing target.
