controller queue interface

Purpose

This page documents controller-queue.html as the queue interface surface in the now-playing system.

Current wiki evidence already supports a stronger statement: controller-queue.html is a real queue surface, it has both standalone and embedded roles, and it is reused by other controller/kiosk flows instead of being a one-off helper fragment.

Scope

What this page now treats as established:

  • controller-queue.html is the main queue inspection/control page
  • controller-queue.html?embedded=1 is the compact embedded version used inside other shells
  • the page participates in kiosk right-pane routing
  • the page is distinct from Queue Wizard

The remaining gap is the exact backend route/module contract behind every action.

High-level role

controller-queue.html is the queue inspection and direct queue interaction surface. It answers what is in the queue, what order it is in, and what queue-focused actions are available from a dedicated page.

Queue interface versus Queue Wizard

controller-queue.html is about queue visibility, direct queue inspection, and queue-focused interaction. Queue Wizard is about queue shaping, preview/apply workflows, and restructuring a queue plan.

Useful shorthand: controller-queue.html = queue interface surface; Queue Wizard = queue planning/shaping surface.

Evidence already established elsewhere

  • Substantive now-playing pages embed controller-queue.html and controller-queue.html?embedded=1.
  • controller-queue.html participates in the embedded-pane contract.
  • Kiosk routing maps kiosk-queue.html to controller-queue.html.

Standalone versus embedded role

In standalone mode, controller-queue.html is the dedicated queue page. In embedded mode, controller-queue.html?embedded=1 acts as a reusable queue subview inside now-playing modals and kiosk right-pane flows.

Queue interface in the broader queue/playback model

Using the established layered model, controller-queue.html is the clearest Layer 3 surface: queue inspection / direct queue UI.

Relationship to now-playing surfaces

The queue interface is tightly tied to the controller-now-playing* family, and queue UI issues may belong to controller-queue.html, its embedded-mode behavior, or its backend queue contract rather than the top-level now-playing shell.

Relationship to kiosk mode

Because kiosk routing maps kiosk-queue.html to controller-queue.html, queue access in kiosk mode is standard controller-backed embedded-pane behavior.

Current status
