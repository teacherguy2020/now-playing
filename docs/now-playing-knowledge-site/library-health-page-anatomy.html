library health page anatomy

Purpose

This page documents the internal anatomy of now-playing/library-health.html.

It is a maintenance workbench with scan and summary actions, animated-art cache inspection, album metadata inspection, performer suggestion/apply workflows, album tag overwrite tools, feat/collaboration cleanup workflows, and moOde library update actions.

Main anatomical regions

  1. shell redirect / page header layer
  2. library scan / summary region
  3. animated-art cache audit region
  4. album metadata inspector
  5. performer suggestion/apply workflow
  6. album tag overwrite tool
  7. feat/collaboration cleanup workflow
  8. moOde library update action
  9. library-health script orchestration layer

Library scan / summary region

Key anchors include #refreshScan, #cards, and #scanStatus. This is the main audit/summary entrypoint for the page.

Animated-art cache audit region

Key anchors include #animatedArtCacheCard, #animatedArtSummary, #animatedArtCacheList, #animatedArtPreview, and the motion/no-motion/all filters. This region summarizes cache state, lists cached items, previews them, and filters by motion state.

Album metadata inspector

Key anchors include #albumPick, #albumInspectBtn, #albumOverviewTab, #albumArtTab, #albumGenreTab, and #albumCleanupTab. This is the richest central inspection region of the page.

Performer suggestion/apply workflow

Key anchors include #albumPerformerInput and #albumApplyPerformersBtn. This is a specialized maintenance workflow, not just a passive data viewer.

Album tag overwrite tool

Key anchors include #albumTagKey, #albumTagValue, #albumTagClearEmpty, #albumTagFillAllBtn, and #albumTagApplyBtn.

Feat/collaboration cleanup workflow

Key anchors include #featAlbumPick, #featScanBtn, #featApplyBtn, #featNormalizeAll, #featStatus, and #featOut.

moOde library update action

Key anchors include #updateMoodeLibraryBtn and #moodeUpdateStatus. This is one of the clearest bridges from local maintenance work into live moOde-side state.

Library-health script orchestration layer

Most page behavior appears to be driven by scripts/library-health.js, which likely orchestrates the scan summary, animated-art cache, album inspection, performer apply workflows, tag overwrite behavior, feat cleanup, and moOde update action.

Practical “where do I start?” map

  • Scan summary: start with #refreshScan, #cards, #scanStatus, and scan logic in scripts/library-health.js.
  • Animated-art cache: start with the animated-art anchors and related logic in scripts/library-health.js.
  • Album metadata inspection: start with the album inspector anchors and inspection logic in scripts/library-health.js.
  • Performer application: start with #albumPerformerInput, #albumApplyPerformersBtn, and performer-related logic.
  • Tag overwrite: start with the album tag anchors and overwrite logic.
  • Feat cleanup: start with the feat cleanup anchors and related logic.
  • moOde library update: start with #updateMoodeLibraryBtn, #moodeUpdateStatus, and moOde update logic.

Current status

This is the eighth real page-anatomy reference in the wiki. Its main job is making explicit that library-health.html is not just a health readout, but a multi-region maintenance workbench.
