api youtube radio and integration endpoints

Purpose

This page documents the API endpoint families most directly associated with feature-specific integrations currently covered in the wiki.

The clearest documented families right now are:

  • YouTube integration endpoints
  • radio metadata debug/evaluation endpoints
  • podcast automation endpoints adjacent to integration work

Auth / protection model

Several of these routes use the same protected app-host pattern already seen elsewhere:

  • track key required for protected operations
  • sent via x-track-key

1. YouTube endpoint family

Documented endpoints:

  • POST /youtube/search
  • POST /youtube/playlist
  • POST /youtube/resolve
  • POST /youtube/queue

This family handles:

  • search
  • playlist expansion
  • metadata resolution
  • queue submission

Current documented queue modes include append, crop, and replace.

2. Radio metadata debug / evaluation family

Documented endpoints:

  • GET /debug/radio-metadata-log?limit=<n>
  • POST /debug/radio-metadata-log/clear

This family exposes radio metadata transformation quality for QA/debug work: raw metadata, parsed metadata, enrichment reasons, and repeated-clear/test cycles.

3. Podcast automation routes adjacent to integration work

Documented endpoints:

  • GET /podcasts/nightly-status
  • POST /podcasts/nightly-run

These are not generic config-control routes, but feature-specific automation routes that belong near the integration branch.

Current status

This page gives the API branch an integration-feature view: external-content ingestion, radio QA/debug, and feature automation routes all now have an API-centered home instead of being documented only through UI pages.
