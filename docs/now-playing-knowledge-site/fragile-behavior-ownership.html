fragile behavior ownership

Purpose

This page maps historically fragile now-playing behaviors to the pages, route modules, helper layers, and local-environment realities that most plausibly own them.

It is the Phase 2 bridge from route ownership truth and playback authority truth into failure-prone engineering reality.

Fragile behavior 1: iframe vs top-level rendering

This path is fragile because visible behavior can differ when a page is loaded top-level, embedded in an iframe, routed through display.html, or hosted inside an app/controller shell.

Main ownership anchors:

  • Pages: index.html, display.html, app.html
  • Runtime/script: scripts/index-ui.js
  • Styling: styles/index1080.css

Typical failure shapes: embedded/top-level background differences, host page blamed instead of final render target, and CSS-only explanations hiding runtime/bootstrap causes.

Fragile behavior 2: Next Up / Alexa-mode path

This path is structurally fragile because it spans controller UI, queue state, /next-up, Alexa was-playing state, YouTube queue hints, and queue-advance behavior.

Main ownership anchors:

  • moode-nowplaying-api.mjs
  • src/routes/queue.routes.mjs
  • src/routes/config.diagnostics.routes.mjs
  • src/services/mpd.service.mjs
  • Alexa-side areas: alexa/, lambda_bundle/, lambda_upload/

Fragile behavior 3: last-good-render preservation

This path is fragile because transient refresh errors can tempt people to clear visible state too aggressively, even when preserving a last-known-good render is the correct guardrail.

Main ownership anchors:

  • scripts/index-ui.js
  • src/routes/art.routes.mjs
  • Companion concept page: playback-authority-by-mode

Fragile behavior 4: display wake / runtime-admin / moOde-host interaction

This path is fragile because it crosses app host logic, runtime-admin routes, SSH-backed moOde host control, and local moOde override scripts.

Main ownership anchors:

  • src/routes/config.runtime-admin.routes.mjs
  • moode-nowplaying-api.mjs
  • integrations/moode/
  • ops/moode-overrides/
  • surfaces like displays.html and kiosk-designer.html

Fragile behavior 5: radio metadata guardrails and enrichment suppression

This path is fragile because raw stream metadata is not safe to treat as final truth. The project already depends on holdback, cleanup, enrichment suppression, station-logo fallback, and cache-backed fallback behavior.

Main ownership anchors:

  • src/routes/art.routes.mjs
  • radio diagnostics/eval surfaces
  • playback-authority-by-mode

Fragile behavior 6: mode-sensitive art/current-song/status truth

This broader fragile path exists because people often assume one universal authority for current song, art, and status. In reality, truth is mode-dependent.

Main ownership anchors:

  • src/routes/art.routes.mjs
  • src/routes/queue.routes.mjs
  • src/routes/config.diagnostics.routes.mjs
  • src/services/mpd.service.mjs
  • playback-authority-by-mode

Ownership pattern across fragile behaviors

  • Fragility is often cross-layer.
  • Route modules often own more than expected.
  • Local runtime reality matters, especially for AirPlay metadata and display wake/moOde behavior.
