app shell anatomy

Purpose

This page documents the internal anatomy of now-playing/app.html.

app.html is both a broad desktop/browser shell and a page with its own substantial shell-side UI regions, cards, controls, modals, and route-backed behavior.

Main anatomical regions

  1. hero transport strip
  2. hero queue card
  3. top tabs / page routing bar
  4. display / Peppy / moOde action controls
  5. status pill strip
  6. iframe-hosted child surface (#appFrame)
  7. queue modal family
  8. shell-to-frame theme / normalization / sizing logic

Hero transport strip

Anchored by #heroTransport and supported by scripts/hero-transport.js. This is one of the first shell-level summaries of playback state.

Hero queue card

This is the most important anatomy discovery: the queue card is a real shell-side queue UI, not just a link to controller-queue.html.

Main DOM anchors:

  • #heroQueueCard
  • #heroQueueWrap
  • #heroQueueNextUp
  • #heroQueueToggle
  • #heroQueueShuffle
  • #heroQueueClear
  • #heroQueueCrop
  • #heroQueueSavePlaylist
  • #heroQueueApplyFilter
  • #heroQueueSavePresets
  • #heroQueueRouteAlexa
  • #heroQueueConsume
  • #heroQueueCrossfade
  • #heroQueueBusyOverlay

Main logic center: async function syncHeroQueue().

It currently does all of these:

  • fetches runtime track key via /config/runtime
  • fetches queue truth via /config/diagnostics/queue
  • fetches playback summary via /now-playing
  • fetches next-up truth via /next-up
  • fetches Alexa was-playing state via /alexa/was-playing
  • handles collapsed vs expanded queue behavior
  • derives and renders Next Up state
  • renders queue rows inside #heroQueueWrap
  • manages rating-filter-based queue behavior

Related modal flows: openQueueAlbumModal(...), openArtistAlbumsModal(...), openQueueTrackMetaModal(...).

If asked to improve the queue card: open app.html, search heroQueue, then read syncHeroQueue() before anything else.

Top tabs / page routing bar

Anchored by #appTabs. This is the main shell routing/navigation region for Config, Diagnostics, Alexa, Library, Queue Wizard, Radio, Podcasts, YouTube, Displays, and Theme.

Display / Peppy / moOde action controls

Buttons adjacent to the tabs include #pushPeppyBtn, #showMoodeUiBtn, #showPlayerBtn, #showVisualizerBtn, and #refreshDisplayBtn. This region crosses the UI/runtime/host boundary.

Status pill strip

Anchored by #statusInline and includes pills like #apiPill, #webPill, #alexaPill, #peppyPill, #moodePill, #notifyPill, and #scrobblePill.

Iframe-hosted child surface

#appFrame is the embedded content region for tab-routed child pages. A new agent should always ask whether a problem is in the shell, the child page, or the shell-frame contract.

Queue modal family

Anchored by #queueAlbumModal, #queueAlbumBody, and #queueAlbumClose. Supports queue-row drill-down behaviors like album details, artist albums, and track metadata.

Shell-to-frame theme / normalization / sizing logic

app.html also owns substantial iframe theming, normalization, and sizing logic via functions like syncFrameWidth(), requestFrameHeightSync(), applyThemeBridgeToFrame(), attachFrameAutoResizeObservers(), and normalizeEmbeddedDoc().

Practical “where do I start?” map

  • Queue card: search heroQueue, read syncHeroQueue(), then the queue modal helpers.
  • Transport strip: start with #heroTransport and scripts/hero-transport.js.
  • Status pills: start with #statusInline and the pill IDs.
  • Tab/page behavior: start with #appTabs and the iframe source/routing logic.
  • moOde/Peppy actions: start with the display action buttons and runtime-admin-related shell logic.

Current status

This is the first real page-anatomy reference in the wiki. Its most important job is making explicit that app.html is not just an iframe shell, and the hero queue card is not just a link to the fuller queue page.
