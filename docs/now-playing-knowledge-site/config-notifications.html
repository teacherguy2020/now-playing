config notifications

Purpose

This page documents the track-notification portion of now-playing/config.html.

It exists because the Config page contains a specific notification feature cluster built around track notifications, a background monitor, Pushover credentials, timing/deduplication tuning, and Alexa-age-related filtering.

Main visible controls

  • featurePushover
  • notifyEnabled
  • pollMs
  • dedupeMs
  • alexaMaxAgeMs
  • pushoverToken
  • pushoverUser

Feature enablement

The feature is derived from whether there is meaningful Pushover config or track-notify state, not just from a single stored boolean.

Background monitor running

notifyEnabled maps to notifications.trackNotify.enabled. This is distinct from top-level feature/provider enablement.

Polling and deduplication tuning

Observed save/load behavior includes:

  • notifications.trackNotify.pollMs (default 3000)
  • notifications.trackNotify.dedupeMs (default 15000)

This means the notification subsystem is poll-driven and explicitly dedupe-aware.

Alexa max age ms

alexaMaxAgeMs maps to notifications.trackNotify.alexaMaxAgeMs (default 21600000). This indicates that notification behavior is cross-wired with Alexa-related recency logic.

Pushover credentials

Observed save/load behavior includes:

  • notifications.pushover.token
  • notifications.pushover.userKey

These are also cached locally under:

  • nowplaying.secret.pushoverToken
  • nowplaying.secret.pushoverUser

Visibility / gating behavior

syncPushoverCardVisibility() suggests that enabling notifications tends to push the UI toward also running the monitor, and that cached credentials are backfilled when appropriate.

Important scope note

Based on current repo-visible evidence, this Config module is specifically a Pushover-backed notification surface. The wiki should not overstate it as a broad multi-provider notification matrix.

User/operator workflow model

  1. Enable notifications
  2. Enter Pushover token and user key
  3. Ensure background monitor is running
  4. Tune poll/dedupe/Alexa-age values
  5. Save config and observe behavior

Current status

This module is best understood as a monitor-driven, timing-tunable, Pushover-backed track notification subsystem.
