route ownership map

Purpose

This page maps major now-playing behaviors to the route modules and bootstrap files that most plausibly own them.

It is the bridge between conceptual pages and code-level ownership.

Ownership model at a glance

  • Top-level page files own user-facing surface behavior.
  • moode-nowplaying-api.mjs owns server/bootstrap assembly.
  • Focused route modules often own real feature behavior.
  • Services and shared libs are support layers, not always the main owners.

Strong current ownership anchors

API bootstrap and registration

moode-nowplaying-api.mjs is the main API bootstrap and route registration point.

Queue and queue-advance ownership

src/routes/queue.routes.mjs is the strongest current route-level owner for queue-head deletion, advance, resolve, prime, and fallback behavior.

Queue Wizard ownership

  • src/routes/config.queue-wizard-basic.routes.mjs
  • src/routes/config.queue-wizard-preview.routes.mjs
  • src/routes/config.queue-wizard-apply.routes.mjs
  • src/routes/config.queue-wizard-vibe.routes.mjs
  • shared support: src/lib/browse-index.mjs

Browse and recent-library ownership

src/routes/config.browse.routes.mjs is the best current anchor for browse and recent-surface behavior.

Runtime-admin and moOde/display control ownership

src/routes/config.runtime-admin.routes.mjs owns runtime config mutation, environment discovery, moOde/browser/display control, and SSH-backed operational side effects.

Diagnostics / playback test ownership

src/routes/config.diagnostics.routes.mjs is the strongest current diagnostics-facing playback/control anchor.

Art ownership

src/routes/art.routes.mjs owns current/track art serving, cache/reuse, resize, blurred backgrounds, and mode-sensitive current-art resolution behavior.

Ratings ownership

src/routes/rating.routes.mjs owns rating reads/writes including /rating/current.

Track serving ownership

src/routes/track.routes.mjs owns /track serving, Alexa-gated access, range support, and optional transcoding/cache behavior.

Controller-profile ownership

src/routes/config.controller-profile.routes.mjs owns controller/mobile/tablet saved profile state.

Config aggregation ownership

src/routes/config.routes.index.mjs is the aggregation hub for many focused config/library-health/queue-wizard route modules.

Supporting service and library ownership

  • src/services/mpd.service.mjs = reusable MPD primitives
  • src/lib/browse-index.mjs = shared browse-index support layer

Ownership by behavior family

  • Direct playback controls: start with config.diagnostics.routes.mjs, mpd.service.mjs, and moode-nowplaying-api.mjs.
  • Queue advance / next-up / queue head: start with queue.routes.mjs.
  • Queue Wizard preview/apply: start with the config.queue-wizard-* route modules plus browse-index.mjs.
  • Controller queue UI backing: start with controller-queue.html, queue.routes.mjs, config.diagnostics.routes.mjs, and bootstrap wiring.
  • Browse/recent/library drill-down data: start with config.browse.routes.mjs and browse-index.mjs.
  • moOde display/runtime/browser target behavior: start with config.runtime-admin.routes.mjs.
  • Art/current-art behavior: start with art.routes.mjs.
  • Ratings/current-song rating: start with rating.routes.mjs.

What is still open

Still not fully locked down: the exact full backend contract behind controller-queue.html, the exact caller-to-route chain for every queue UI action, the full route inventory for direct playback actions beyond current diagnostics-heavy anchors, and some broader radio/playlist ownership areas.

Current status

This page gives the wiki a needed bridge from concepts to code ownership and from surfaces to route modules.
