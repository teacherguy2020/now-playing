kiosk interface

Purpose

This page documents the kiosk branch of the now-playing ecosystem.

A stronger current interpretation is:

  • kiosk is a presentation/shell mode
  • it is not just another renderer alongside Player/Peppy/Visualizer
  • the live kiosk path is largely controller-backed
  • kiosk.html is an entry bridge, not the live shell
  • many kiosk-*.html pages are redirect aliases into controller-family pages
  • controller-kiosk.html is a separate scaffold/prototype path

What kiosk is in the current repo

Kiosk is the presentation shell built around a controller-backed 1280×400-style experience. The entry path resolves profile/theme/recent-source settings and hands off into controller.html, which then hosts controller-family child pages, especially through the kiosk right pane.

Strong current branch map

1. Kiosk entry and redirect handoff

kiosk-launch-and-routing now proves from file inspection that kiosk.html is a profile-sync + redirect bridge into controller.html?kiosk=1&preview=1&rev=....

2. Controller-backed live kiosk shell

controller-kiosk-mode now proves from file inspection that controller.html explicitly detects kiosk mode/editor mode and owns the kiosk layout shell, pane iframe, and child-route remapping.

3. Kiosk child-route alias layer

The current direct file-backed truth is that many kiosk-*.html pages are pure redirect aliases into controller pages.

4. Right-pane and embedded child behavior

kiosk-right-pane-routing and embedded-pane-contracts document that kiosk is also a parent/child navigation system, not just a shell theme.

5. Kiosk editor/designer paths

kiosk-designer, kiosk-editor-mode, and controller-kiosk-mode cover the distinct editor/designer branch.

6. Kiosk anatomy as a shell path

kiosk-shell-anatomy treats kiosk as an anatomical shell path, not a single page.

What this branch is now confident about

  • kiosk is a presentation/shell mode, not a display renderer
  • kiosk.html is a profile-sync + redirect bridge into controller.html
  • controller.html owns real kiosk shell behavior
  • many kiosk-named child pages are redirect aliases into controller-family pages
  • right-pane and embedded child-page behavior are core kiosk mechanics
  • controller-kiosk.html is a scaffold/prototype path, not the main live kiosk entry

Current status

This page should now be read as the parent hub for the kiosk branch. The current wiki supports a much stronger model: kiosk is a controller-backed shell mode, and the entry bridge, live shell, right-pane behavior, alias pages, and scaffold path are explicitly separated.
