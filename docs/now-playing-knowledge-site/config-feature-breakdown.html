config feature breakdown

Purpose

This page documents the major feature areas inside now-playing/config.html.

It exists because config-interface.md establishes what the Config page is and how it behaves as a surface, but it does not yet decompose the major feature modules inside that page.

Why this page matters

config.html is not a single-purpose settings form. It is a broad runtime/operator console that combines host/runtime setup, feature toggles, credentials/secrets, integration-specific config, service-adjacent controls, diagnostics-adjacent maintenance actions, and advanced raw JSON editing.

Major modules inside config.html

1. Network & Runtime

The foundational setup card and effective gate for much of the rest of the page, including API host/API port/UI port, MPD host/port, moOde SSH host/user, moOde base URL override, track key, music library root, moOde USB mount path, API-host mount base path, and runtime checks.

2. Podcasts

Feature enablement plus podcast-root and nightly-cron helper behavior.

3. Radio Artwork

Feature flag for radio per-track artwork behavior in now-playing flows.

4. moOde Display Enhancement

Controls whether moOde display enhancement / takeover behavior is enabled.

5. mpdscribble controls

Service-adjacent controls for refresh/start/stop plus status visibility.

6. Last.fm / Vibe

Feature enablement and configuration for Last.fm API key, username, recents-replace-radio behavior, and row modes like toptracks, recenttracks, topartists, and topalbums.

7. Alexa setup

Provisioning/setup layer for enablement, public domain, domain health checking, and route webhook URL. This is distinct from the dedicated alexa-interface.md correction/review page.

8. Ratings setup

Feature enablement plus ratings DB operations including sticker DB check, backup listing, backup creation, and backup restore.

9. Track Notifications / Pushover

Current push-notification cluster, including Pushover enablement, background monitor enablement, poll ms, dedupe ms, Alexa max age ms, Pushover token, and Pushover user key.

10. Animated Art

Toggle for animated Apple Music artwork use when available.

11. Album Personnel

Enable/disable the personnel block on the Now Playing page, with host dependency on metaflac.

12. Advanced JSON

Full raw config editing via format/save JSON controls.

Important runtime/write behavior

Repo-visible code indicates that config.html loads runtime config into the form, caches certain secrets locally, persists feature state back into runtime config, and performs feature-specific maintenance calls.

Feature relationships worth keeping explicit

  • Push notifications vs Alexa: related but not the same feature; Config exposes Pushover credentials and track-notify behavior separately from Alexa domain/webhook setup and the separate Alexa corrections/review page.
  • Last.fm vs mpdscribble: strongly related but still distinct; mpdscribble is the scrobble/runtime service layer, while Last.fm is the downstream feature/integration layer used for Vibe and row presentation.
  • Ratings vs general preferences: ratings setup includes persistent database handling and backup/restore workflows.

Architectural interpretation

A good current interpretation is that config.html is really a collection of feature modules, not one monolithic settings form. Some modules are pure feature toggles, some are integration setup surfaces, some are operational maintenance tools, and some bridge directly into live runtime or host-side behavior.

Relationship to other pages

This page should stay linked with config-interface.md, configuration-and-diagnostics-interfaces.md, integrations.md, alexa-interface.md, and diagnostics-interface.md.

At the moment, this page gives config.html a much more honest representation. It shows that Config includes at least these major feature clusters: runtime/network setup, podcasts, radio artwork, moOde display enhancement, mpdscribble, Last.fm / Vibe, Alexa setup, ratings DB management, Pushover track notifications, animated art, album personnel, and advanced raw JSON editing.
