config feature breakdown

Purpose

This page documents the major feature areas inside now-playing/config.html.

It exists because config-interface.md establishes what the Config page is and how it behaves as a surface, but it does not yet decompose the major feature modules inside that page.

This page is the feature-level breakdown for config.html.

Why this page matters

config.html is not a single-purpose settings form. It is a broad runtime/operator console that combines:

  • host/runtime setup
  • feature toggles
  • credentials/secrets
  • integration-specific config
  • service-adjacent controls
  • diagnostics-adjacent maintenance actions
  • advanced raw JSON editing

Major modules inside config.html

1. Network & Runtime

Foundational host/path/runtime setup, including API host, UI port, MPD host/port, moOde SSH host/user, track key, music roots, and path checks.

2. Podcasts

Feature enablement plus podcast-root and nightly-cron helper behavior.

3. Radio Artwork

Feature flag for radio per-track artwork behavior in now-playing flows.

4. moOde Display Enhancement

Controls whether moOde display enhancement / takeover behavior is enabled.

5. mpdscribble controls

Service-adjacent controls for refresh/start/stop plus status visibility.

6. Last.fm / Vibe

Feature enablement and configuration for:

  • Last.fm API key
  • username
  • recents-replace-radio behavior
  • row mode (toptracks, recenttracks, topartists, topalbums)

This is the main scrobbling/Vibe-oriented config cluster.

7. Alexa setup

Provisioning/setup layer for:

  • enablement
  • public domain
  • domain health checking
  • route webhook URL

This is distinct from the dedicated alexa-interface.md correction/review page.

8. Ratings setup

Feature enablement plus ratings DB operations:

  • check sticker DB
  • list backups
  • create backup
  • restore selected backup

This is a real maintenance subsystem, not just a toggle.

9. Track Notifications / Pushover

Current push-notification cluster, including:

  • Pushover enablement
  • background monitor enablement
  • poll ms
  • dedupe ms
  • Alexa max age ms
  • Pushover token
  • Pushover user key

So yes: push notifications are covered here, and the current visible provider is Pushover.

10. Animated Art

Toggle for animated Apple Music artwork use when available.

11. Album Personnel

Enable/disable the personnel block on the Now Playing page, with host dependency on metaflac.

12. Advanced JSON

Full raw config editing via format/save JSON controls.

Important feature relationships

  • Last.fm vs mpdscribble: related, but distinct — mpdscribble is the scrobble/service side, Last.fm is the discovery/Vibe integration side.
  • Alexa setup vs Alexa corrections: Config owns enablement/domain/webhook; alexa.html owns corrections/review.
  • Ratings: not just a preference toggle — includes DB stewardship.
  • Push notifications: currently surfaced here as a Pushover + track-monitoring module.

Important API/maintenance surfaces visible from Config

  • POST /config/alexa/check-domain
  • /config/ratings/sticker-status
  • /config/ratings/sticker-backups
  • /config/ratings/sticker-backup
  • /config/ratings/sticker-restore
