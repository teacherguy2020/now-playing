queue interface vs Queue Wizard

Purpose

This page exists to answer another recurring confusion quickly:

Am I looking at the queue interface, or am I looking at Queue Wizard?

The answer matters because these are related but not interchangeable parts of the system.

The short answer

  • Queue interface = viewing and directly interacting with the current queue
  • Queue Wizard = previewing, shaping, and curating what the queue should become

If you are changing the current queue list UI, row controls, or embedded queue surface, start with controller-queue-interface.

If you are changing preview/apply behavior, album-level queue planning, or higher-order queue generation, start with queue-wizard-internals.

Key proof points

1. Controller queue is the queue surface

controller-queue-interface says:

  • controller-queue.html is the main queue inspection/control surface
  • it has standalone and embedded roles
  • it appears in now-playing and kiosk flows

2. Queue Wizard is the queue-shaping layer

queue-wizard-internals says:

  • Queue Wizard is the queue-shaping and queue-curation layer
  • it sits above raw queue mutation
  • it uses preview/apply semantics
  • it is important for album-level and feature-driven queue planning

3. The model page separates queue control from queue shaping

queue-and-playback-model distinguishes:

  • queue control = inspect, append, replace, crop, load items
  • queue shaping = previewing and restructuring what should be played

That is the conceptual split behind the page split.

Practical starting paths

  1. controller-queue-interface

If the queue preview/apply flow is wrong

  1. queue-wizard-internals
  2. queue-and-playback-model

If you are not sure which one you mean

  1. queue-and-playback-model
  2. Then choose either controller-queue-interface or queue-wizard-internals

Do not confuse

  • Queue interface is not the same thing as Queue Wizard.
  • Embedded queue UI is not the same thing as queue planning.
  • Raw queue mutation is not the same thing as queue shaping.

  • Queue Wizard = the higher-order preview/apply queue-shaping layer
