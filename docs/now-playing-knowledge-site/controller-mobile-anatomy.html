controller mobile anatomy

Purpose

This page documents the internal anatomy of now-playing/controller-mobile.html.

It is a compact controller shell with its own layout priorities, interaction flow, and region-level responsibilities.

Main anatomical regions

  1. compact now-playing header region
  2. integrated transport controls
  3. quick search strip
  4. library/category list
  5. recent-content region
  6. queue and queue-adjacent entrypoints
  7. modal/workflow regions
  8. mobile layout/state orchestration

Compact now-playing header region

Current visible anchors suggest a familiar controller-family structure including #npCard, #npArtWrap, #art, #line1, #line2, and #line3. This is the mobile shell’s compact playback summary region.

Integrated transport controls

The mobile shell includes integrated playback controls rather than outsourcing transport behavior entirely. Current anchors suggest transport clusters such as #npCtrls, #phone-controls, and data-np-act.

Quick search strip

Anchored by #homeQuickSearch, #homeQuickResults, and .quickWrap. This is the mobile shell’s fast entrypoint into browse/search/jump behavior.

Library/category list

Anchored by #libraryList, row-level structures such as data-row-target, and wrapped row behavior such as data-row-wrap. This appears to provide the main browse-entry structure for mobile.

Recent-content region

Appears to include anchors like #recentTitle, #recentRail, #recentPeekRail, and #railModeBtn. This region likely provides recent-content scanning and quick re-entry into recent items.

Queue and queue-adjacent entrypoints

The mobile shell appears to contain queue-adjacent entrypoints through list/category row targets, queue-related wrapped rows or actions (data-queue-act), references to queue routes and queue-wizard routes, and likely entry into fuller queue surfaces such as controller-queue.html.

Modal/workflow regions

Current visible anchors suggest mobile shell workflow/modal regions overlapping with the controller family, including things like #audioInfoModal, #recentAlbumModal, #recentArtistModal, and queue/album-related modal structures.

Mobile layout/state orchestration

controller-mobile.html appears to contain page-level orchestration for compact layout, search/list/recent interplay, and queue/browse/navigation behavior in reduced space.

Practical “where do I start?” map

  • Compact now-playing area: start with #npCard, #npArtWrap, #art, and the line fields.
  • Mobile transport controls: start with #npCtrls, #phone-controls, and data-np-act.
  • Quick search: start with #homeQuickSearch, #homeQuickResults, and .quickWrap.
  • Category-row navigation: start with #libraryList and row target/wrap behavior.
  • Recent-content scanning: start with #recentRail, #recentPeekRail, and #railModeBtn.
  • Mobile queue behavior: start with queue-related rows or quick actions in controller-mobile.html, then move to controller-queue.html if the request really belongs to the fuller queue surface.

Current status

This is the fifth real page-anatomy reference in the wiki. Its main job is making explicit that controller-mobile.html is not just a smaller tablet page, but a compact controller shell with its own region-level responsibilities.
