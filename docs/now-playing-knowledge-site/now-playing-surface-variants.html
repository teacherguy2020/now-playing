now-playing surface variants

Purpose

This page documents the controller-now-playing* family of HTML surfaces in the now-playing ecosystem.

This family is not a pile of equal peers. Current repo truth shows a mix of substantive implementation pages, device/layout variants, thin redirect shims, and one alias compatibility shim.

Files in the family

  • controller-now-playing.html
  • controller-now-playing-tablet.html
  • controller-now-playing-grid.html
  • controller-now-playing-ipad.html
  • controller-now-playing-iphone.html
  • controller-nowplaying-iphone.html

Family classification

1. Main implementation page: controller-now-playing.html

controller-now-playing.html is one of the main now-playing implementation centers in the repo. It is a large substantive controller surface with integrated now-playing UI, modal infrastructure, embedded fallback handling, transport controls, and iphone=1 behavior inside the same file.

2. Tablet implementation variant: controller-now-playing-tablet.html

This is a substantive implementation page, not a redirect shim.

3. Grid implementation variant: controller-now-playing-grid.html

This is also a substantive implementation page, not a wrapper.

4. iPad redirect shim: controller-now-playing-ipad.html

Pure redirect shim to /controller-now-playing-tablet.html, preserving query parameters.

5. iPhone redirect shim: controller-now-playing-iphone.html

Pure redirect shim to controller-now-playing.html?iphone=1.

6. Alias shim: controller-nowplaying-iphone.html

Simple redirect alias to controller-now-playing-iphone.html.

Working family model

  • Substantive implementation pages: controller-now-playing.html, controller-now-playing-tablet.html, controller-now-playing-grid.html
  • Redirect / alias pages: controller-now-playing-ipad.html, controller-now-playing-iphone.html, controller-nowplaying-iphone.html

Shared behaviors across the substantive variants

Across the three substantive pages, the repo shows recurring patterns such as background art/blur presentation, now-playing text and art rendering, embedded-mode fallback handling, queue modal infrastructure, album modal infrastructure, and controller-style visual language.

Important variant differences

controller-now-playing.html is the strongest mainline implementation and also carries the iPhone-specialized path through ?iphone=1. controller-now-playing-tablet.html is the tablet-oriented variant. controller-now-playing-grid.html is the grid/phone-oriented substantive variant.

Relationship to kiosk and embedded behavior

This family intersects directly with the kiosk/embedded branch because the substantive pages include embedded-mode handling and modal/child-surface behavior that can be used in controller-hosted or kiosk-hosted flows.

Anatomy companion page

controller-now-playing-anatomy

Use it when the real question is not just which variant this is, but which region inside controller-now-playing.html owns the thing you need to change.

What this page is now confident about

  • the now-playing family has three substantive implementation pages
  • the iPad and iPhone named pages are redirect shims, not main implementation pages
  • controller-now-playing.html carries the iPhone mode through ?iphone=1
  • controller-nowplaying-iphone.html is just an alias shim
  • the family shares real behavioral DNA around art, modals, embedded handling, and controller-side now-playing presentation

Current status

This page now gives the now-playing family a firmer structural map: substantive implementation pages versus shims, main page versus tablet/grid variants, iPhone mode as a query-param branch of controller-now-playing.html, and alias pages as actual redirects rather than pseudo-surfaces.
