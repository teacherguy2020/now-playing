# Diagnostics

Purpose: inspect live state and safely trigger playback/queue actions.

## Covers
- `diagnostics.html`
- `/config/diagnostics/*` endpoints

## Focus areas
- queue state (`headPos`, items)
- playback actions (`play/pause/next/prev/random/repeat`)
- endpoint explorer + test payloads

## Notes
- Requires `x-track-key` for write actions.
